# Marker Swift ‚Äî Development Context

## What This Is

Native macOS hybrid app: Swift/AppKit shell + WKWebView hosting Milkdown Crepe editor.
Replaces the Tauri version at `/Users/esaipetch/devwork/marker` (archived).

## Two-Repo Relationship

- **`/Users/esaipetch/devwork/marker`** ‚Äî ARCHIVED Tauri app. **Source of truth for editor TypeScript only.** JS changes are made here, rebuilt via Vite, and copied to marker-swift.
- **`/Users/esaipetch/devwork/marker-swift`** ‚Äî The native Swift app. All UI is native AppKit. The Vite webview bundle (`marker/dist-editor/`) lives at `Marker/Resources/`.

**Files in `marker/` that are webview-relevant:**

**Architecture (LOCKED ‚Äî do not change these decisions):**
- **Single WKWebView** ‚Äî TabPool.ts manages multiple Crepe instances via display:none/block
- **Custom tab bar** (NSView subclass) ‚Äî NOT NSWindow.tabbingMode
- **Swift‚ÜíJS:** `webView.callAsyncJavaScript("await marker.xxx(args)", arguments: [...], in: nil, in: .page)` ‚Äî all marker.* functions are async; `evaluateJavaScript` fails with "unsupported type" on Promises. Use `evaluateJavaScript` only for sync DOM queries.
- **JS‚ÜíSwift:** `window.webkit.messageHandlers.marker.postMessage({type, ...})`
- **getMarkdown uses callback pattern** ‚Äî Swift calls requestMarkdown, JS posts back via postMessage
- **Separate EditorMessageHandler class** ‚Äî avoids WKUserContentController retain cycle (holds VC weakly)

## Hard-Won Lessons (DO NOT repeat these mistakes)

1. **`@main` on NSApplicationDelegate does NOT call NSApplication.run() without a storyboard.** Must use `main.swift` with explicit `NSApplication.shared.run()`.

2. **xcodegen `resources:` config doesn't reliably copy files.** Use `postCompileScripts` to `cp -R` resources.

3. **`Bundle.main.url(forResource:withExtension:subdirectory:"Resources")` is WRONG.** Use no subdirectory parameter.

4. **editor.js is an IIFE, not an ES module.** No `type="module"` on the script tag.

5. **CSS filename contains a content hash** (e.g., `style-Bw5clF0G.css`). After rebuilding, update the `<link>` href in editor.html.

6. **`NSApp.setActivationPolicy(.regular)` is required** for programmatic AppKit apps.

7. **editor.html CSP must include `'unsafe-eval'`.** Milkdown/ProseMirror requires eval.

8. **editor.html must call `marker.init()` after the script loads.** Without this, the editor is blank.

9. **editor.html is manually maintained, NOT generated by Vite.**

10. **`windowDidLoad` is NOT called for programmatically created windows.** Use explicit setup after `showWindow`.

11. **`evaluateJavaScript` fails on async JS functions** returning Promises ("unsupported type"). Always use `callAsyncJavaScript` for async marker.* calls.

12. **`NSColor.cgColor` snapshots are static.** Use `viewDidChangeEffectiveAppearance` or `AppearanceAwareView` to re-snapshot on theme change.

13. **`NSSearchField` action fires on Enter only, not per-keystroke.** Use `controlTextDidChange` delegate method for live search.

14. **`isHidden = true` does NOT collapse height in manual Auto Layout.** Use an explicit zero-height constraint that's toggled on show/hide.

15. **Tab IDs must be unique.** Use `UUID().uuidString`, not millisecond timestamps (collide on rapid opens).

## Editor Bundle Rebuild Workflow

When you change any TypeScript in `marker/`:
1. `cd /Users/esaipetch/devwork/marker`
2. `npx vite build --config vite.config.webview.ts`
3. `cp -R dist-editor/* /Users/esaipetch/devwork/marker-swift/Marker/Resources/`
4. **Check if CSS hash changed:** `ls dist-editor/assets/style-*.css` ‚Äî update `<link href>` in editor.html if needed
5. `cd /Users/esaipetch/devwork/marker-swift && xcodegen generate`
6. `xcodebuild -project Marker.xcodeproj -scheme Marker -configuration Debug build`

## Bridge API

**Swift ‚Üí JS (callAsyncJavaScript):**
- `marker.init()` ‚Äî creates TabPool, opens welcome tab, posts "ready"
- `marker.openTab(tabId, markdown)` ‚Äî opens/switches to tab
- `marker.switchTab(tabId, markdown)` ‚Äî shows existing tab (markdown used if evicted)
- `marker.requestMarkdown(tabId)` ‚Äî triggers postMessage callback with content
- `marker.closeTab(tabId)` ‚Äî destroys tab
- `marker.scrollToHeading(tabId, index)` ‚Äî scroll to nth heading
- `marker.setTheme("dark"|"light")` ‚Äî change theme
- `marker.setFontSize(px)` / `marker.setFontFamily(family)`
- `marker.find(tabId, query, caseSensitive, wholeWord, useRegex)` ‚Äî search, returns {count, currentIndex}
- `marker.findNext(tabId)` / `marker.findPrev(tabId)` ‚Äî cycle matches
- `marker.replaceOne(tabId, replacement, query, ...)` ‚Äî replace current match
- `marker.replaceAllMatches(tabId, query, replacement, ...)` ‚Äî replace all
- `marker.clearSearch(tabId)` ‚Äî remove highlights
- `marker.insertText(tabId, text)` ‚Äî insert text at cursor

**JS ‚Üí Swift (postMessage):**
- `{type: "ready"}` ‚Äî editor initialized
- `{type: "dirty", tabId, isDirty}` ‚Äî content changed
- `{type: "cursorChanged", tabId, line, col}` ‚Äî cursor moved
- `{type: "markdown", tabId, content}` ‚Äî response to requestMarkdown
- `{type: "evicted", tabId, markdown}` ‚Äî tab evicted from LRU pool
- `{type: "imagePaste", tabId, base64, extension}` ‚Äî image pasted/dropped

## Build & Run

```bash
cd /Users/esaipetch/devwork/marker-swift

# Regenerate Xcode project (after changing project.yml)
xcodegen generate

# Build
xcodebuild -project Marker.xcodeproj -scheme Marker -configuration Debug build

# Install to /Applications

open /Applications/Marker.app
```

## Known Issues / Future Work

- **No `validateMenuItem` for all items** ‚Äî some menu items could be smarter about disabled state
- **Outline heading indent** uses fragile constraint lookup on reused cells
- **Word count** queries DOM on every keystroke ‚Äî consider debouncing
